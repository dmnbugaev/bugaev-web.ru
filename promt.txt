## Задача: Заменить существующую форму на прямую интеграцию с Formspree

### Проблема текущей реализации
Сложная логика с параллельными запросами и отсутствием обработки CORS приводит к ненадежной работе формы.

### Новая реализация

1. **Упростить `composables/useFormSubmit.ts`**:
   - Удалить все запросы к Make.com из клиентского кода
   - Использовать стандартную Formspree интеграцию через `fetch` с правильными заголовками
   - Добавить режим "fire-and-forget" для Make.com через серверный прокси или отказаться от него

2. **Создать простой прокси для Make.com** (если нужно сохранить Telegram уведомления):
   - Создать `server/api/make-webhook.post.ts`
   - Проксировать запросы на Make.com без ожидания ответа

3. **Обновить форму**:
   - Добавить скрытое поле `_gotcha` для защиты от ботов
   - Использовать `formspree.io` напрямую через `fetch`
   - Добавить `mode: 'no-cors'` как fallback

4. **Упростить валидацию**:
   - Оставить только клиентскую валидацию
   - Убрать двойную отправку

### Ключевые изменения в коде
- Минимальные изменения, только исправление проблем с отправкой
- Удаление параллельных запросов
- Добавление обработки сетевых ошибок с понятными сообщениями
- Сохранение всей существующей функциональности (UTM, Метрика)

### Тестирование
После замены проверить:
- Отправку формы при включенном AdBlock
- Отправку при медленном интернете
- Отправку при недоступности Make.com
- Работу Яндекс.Метрики
