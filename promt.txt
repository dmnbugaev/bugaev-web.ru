РОЛЬ: Опытный frontend-разработчик (Nuxt.js/Vue.js) и специалист по верстке (CSS/HTML).

ЗАДАЧА: Оптимизировать мобильную версию сайта на Nuxt.js. Необходимо исправить проблему со слайдером (полоса прокрутки), который в мобильной версии сдвигает все элементы сайта вправо, создавая некрасивый горизонтальный сдвиг и пустое пространство.

ТРЕБОВАНИЯ:
1. Сделать полосу прокрутки (слайдер) прозрачной.
2. Обеспечить, чтобы полоса прокрутки не влияла на ширину контейнеров и не сдвигала элементы верстки. Полоса должна накладываться поверх контента, а не занимать место в макете.

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ:
- В мобильной версии (и в целом для всех устройств, если применимо) скроллбар остается функциональным, но визуально невидим или полупрозрачен.
- Ширина блоков (`width: 100%`) остается равной ширине окна браузера, даже если появляется стандартная полоса прокрутки. Контент не дергается при открытии/закрытии модальных окон или навигации.

РЕШЕНИЕ (ИНСТРУКЦИЯ ДЛЯ РАЗРАБОТЧИКА):

1.  **Глобальные CSS стили (для переопределения стандартного скролла):**
    Необходимо добавить в глобальные стили проекта (например, `assets/css/global.css` или в секцию `style` в `app.vue`) следующий код:

    ```css
    /* Для WebKit браузеров (Chrome, Safari, Edge) */
    ::-webkit-scrollbar {
      width: 0;          /* Полная прозрачность и отсутствие места на десктопе */
      background: transparent;
    }

    /* Для Firefox */
    * {
      scrollbar-width: thin; /* Тонкая полоса */
      scrollbar-color: rgba(0, 0, 0, 0.2) transparent; /* Прозрачный фон дорожки */
    }

    /* Опционально: если нужно оставить тонкую полосу, но поверх контента,
       и при этом убрать смещение (только для body) */
    html {
      overflow-x: hidden;
      margin-right: calc(-1 * (100vw - 100%)); /* Классический хак от смещения */
    }

    body {
      overflow-x: hidden;
      width: 100%;
    }
